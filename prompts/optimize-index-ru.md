# OPTIMIZE — сокращение index.md или log.md

Ты — non-interactive агент. Твоя задача: сократить передаваемый markdown-файл до указанного размера, сохранив смысл и структуру.

**Никаких вопросов пользователю. Никакого разговорного тона. Никаких комментариев до и после результата.**

```
%%CURRENT_CONTENT%%
```

## Правила сокращения

### Для `index.md`
- Сохрани **все wikilinks** `[[wiki/...]]` — они структурные, не трогай
- Сохрани **все заголовки** (`## Сущности`, `## Концепции` и т.п.)
- **Сокращай только описания** после тире: «— микрофреймворк для веб-разработки с Tailwind CDN и Alpine.js» → «— микрофреймворк»
- Идеал: 1 строка = `[[link]] — короткое описание в 2-4 слова`
- Порядок строк и их количество менять нельзя

### Для `log.md`
- Сохрани заголовок файла (первые строки до первой записи `## [`)
- Для каждой записи `## [YYYY-MM-DD ...] <тип> | <заголовок>`:
  - Заголовок записи НЕ меняй
  - Содержимое (буллеты) сокращай: убирай повторы, очевидное, лишние детали
  - Старайся сжать до 3-5 главных буллетов на запись
- Порядок записей сохрани

## Формат вывода

Выведи **ТОЛЬКО** новый текст файла. Без обёрток. Без markdown-блоков ```. Без пояснений «Вот результат:». Без финальной сводки.

Первый символ твоего вывода = первый символ нового файла.
Последний символ твоего вывода = последний символ нового файла.

Это критично — программа будет сохранять твой stdout как новый файл 1:1.
