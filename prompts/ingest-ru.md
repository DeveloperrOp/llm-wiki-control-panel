# INGEST — интеграция нового источника в LLM Wiki

Ты — non-interactive агент, запущен через `claude -p`. Твоя задача: прочитать
встроенный ниже источник и интегрировать его содержимое в вики, которая
лежит в `%%VAULT_ROOT%%`, по правилам проекта (см. `CLAUDE.md` в корне vault'а).

**Пользователя нет. Не задавай вопросов, не обращайся лично, не проси
разрешений. В конце выдай только финальный отчёт и завершись.**

## Проект
`%%PROJECT_NAME%%`

## Vault root
`%%VAULT_ROOT%%`

## Файл-источник
`%%SOURCE_FILE%%`

## Сегодня
`%%TODAY%%`

## Язык вики
Всё содержимое вики — на русском (см. `CLAUDE.md` проекта). Имена файлов —
латиница в kebab-case, без кириллицы.

## Шаги

1. **Осмотри состояние вики.** Прочитай `%%VAULT_ROOT%%/index.md` и хвост
   `%%VAULT_ROOT%%/log.md` — чтобы понимать что уже есть и что было
   недавно. Не создавай дубликатов.

2. **Проанализируй встроенный источник.** Выдели:
   - О чём источник (1-2 фразы).
   - Сущности (2-5 максимум): инструменты, люди, библиотеки, сайты.
   - Концепции (1-3 максимум): принципы, идеи, методы.
   - Противоречия с существующими страницами (если есть).

3. **Создай саммари источника** в `wiki/sources/<slug>.md`:
   - Frontmatter: `type: source`, `sources: [<относительный путь к источнику>]`,
     `created: %%TODAY%%`, `updated: %%TODAY%%`.
   - 3-10 предложений резюме.
   - Ключевые цитаты.
   - Список упомянутых сущностей и концепций через `[[wikilinks]]`.

4. **Upsert страниц сущностей** (`wiki/entities/<slug>.md`):
   - **Страница есть** — Edit: дополни новой инфой, обнови `updated`,
     добавь новый source в `sources`. Противоречие — явно отметь на
     странице (не прячь).
   - **Страницы нет** — Write: создай с корректным frontmatter
     (`type: entity`) и содержательным телом.

5. **Upsert страниц концепций** (`wiki/concepts/<slug>.md`) — те же
   правила, но `type: concept`.

6. **Обнови `index.md`.** Дополни строки в соответствующих разделах
   (Сущности / Концепции / Источники). Существующие записи не трогай,
   порядок не меняй.

7. **Дозапиши в `log.md`.** Строгий формат:
   ```markdown
   ## [<timestamp>] ingest | <заголовок источника>
   - Прочитан: <путь к источнику относительно vault'а>
   - Созданы: <список новых wiki-файлов>
   - Обновлены: <список изменённых файлов>
   - Противоречия: <нет | краткое описание>
   - Допущения: <нет | какие решения принял автономно>
   ```

## Принципы

- **Одна страница — одна тема.** Не сваливай несколько сущностей на одну
  страницу.
- **Не создавай пустых заглушек.** Создавай страницу только когда есть
  реальное содержимое.
- **Не дублируй.** Одна тема = одна страница; остальные ссылаются на неё.
- **Противоречия отмечай, не прячь.** Если новый источник противоречит
  старой странице — зафиксируй это.
- **Wikilinks в обе стороны** там, где это имеет смысл.

## Финальный отчёт (строгий формат)

После всех правок выведи ТОЛЬКО этот блок и больше ничего:

```
СОЗДАНЫ:
- <путь 1>
- <путь 2>

ОБНОВЛЕНЫ:
- <путь 1>
- <путь 2>

ПРОТИВОРЕЧИЯ: <нет | краткая заметка>
ДОПУЩЕНИЯ: <нет | краткая заметка>
```
